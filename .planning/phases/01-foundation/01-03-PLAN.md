**calcTotals implementation:**
Use a single reduce over the transactions array. The accumulator is initialized with {balance:0, income:0, expenses:0}. For each transaction: if type === 'income', add amount to both income and balance. If type === 'expense', add amount to expenses and subtract from balance. Return a new object each iteration (immutable pattern — do not mutate the accumulator in-place):
```typescript
export function calcTotals(transactions: Transaction[]): SummaryTotals {
  return transactions.reduce<SummaryTotals>(
    (acc, t) => {
      if (t.type === 'income') {
        return { ...acc, income: acc.income + t.amount, balance: acc.balance + t.amount };
      }
      return { ...acc, expenses: acc.expenses + t.amount, balance: acc.balance - t.amount };
    },
    { balance: 0, income: 0, expenses: 0 }
  );
}
```

**groupByCategory implementation:**
Filter to expense-only FIRST (this is documented behavior — income never appears in the spending breakdown chart). Then aggregate by category using a Map<string, number>. Map over the result to add getCategoryMeta labels and colors. Returns CategoryChartPoint[]:
```typescript
export function groupByCategory(transactions: Transaction[]): CategoryChartPoint[] {
  const expenseOnly = transactions.filter((t) => t.type === 'expense');
  const totals = new Map<string, number>();
  for (const t of expenseOnly) {
    totals.set(t.category, (totals.get(t.category) ?? 0) + t.amount);
  }
  return Array.from(totals.entries()).map(([category, value]) => {
    const meta = getCategoryMeta(category as Category);
    return { name: meta.label, value, color: meta.color };
  });
}
```

**formatChartData implementation (highest-risk function — Recharts crash prevention):**
This is the most critical utility in Phase 1. Every output object MUST have all four keys initialized. The invariant is enforced by initializing each month's accumulator with {income:0, expenses:0, balance:0} — the `?? { income: 0, expenses: 0, balance: 0 }` pattern guarantees this.

The balance field is CUMULATIVE (running total across months, not per-month net). This produces a "balance trend" line chart that shows account growth over time.

Sort transactions chronologically before grouping (sort a copy, never mutate the input array). Group by "MMM yyyy" month key using date-fns format + parseISO. Return ChartDataPoint[]:
```typescript
import { format, parseISO } from 'date-fns';
import type { Transaction, SummaryTotals, CategoryChartPoint, ChartDataPoint } from '../types';
import { getCategoryMeta } from '../constants/categories';
import type { Category } from '../types';

export function formatChartData(transactions: Transaction[]): ChartDataPoint[] {
  if (transactions.length === 0) return [];

  const monthMap = new Map<string, { income: number; expenses: number; balance: number }>();

  const sorted = [...transactions].sort(
    (a, b) => new Date(a.date).getTime() - new Date(b.date).getTime()
  );

  for (const t of sorted) {
    const monthKey = format(parseISO(t.date), 'MMM yyyy');
    const existing = monthMap.get(monthKey) ?? { income: 0, expenses: 0, balance: 0 };

    if (t.type === 'income') {
      monthMap.set(monthKey, {
        ...existing,
        income: existing.income + t.amount,
        balance: existing.balance + t.amount,
      });
    } else {
      monthMap.set(monthKey, {
        ...existing,
        expenses: existing.expenses + t.amount,
        balance: existing.balance - t.amount,
      });
    }
  }

  return Array.from(monthMap.entries()).map(([date, values]) => ({
    date,
    income: values.income,
    expenses: values.expenses,
    balance: values.balance,
  }));
}
```

**formatCurrency implementation:**
Use Intl.NumberFormat with 'en-US' locale, 'currency' style, 'USD' currency, minimumFractionDigits:2, maximumFractionDigits:2. Accept number | null | undefined — null/undefined coerce to 0 via nullish coalescing. The browser native API handles zero (returns "$0.00" not "-$0.00"), negative (-$150.00), and large numbers ($1,234.56) correctly without custom logic.

Do NOT use: toFixed(), string concatenation, Intl with accounting signDisplay (returns parentheses not minus sign), custom negative handling.

```typescript
export function formatCurrency(amount: number | null | undefined): string {
  const value = amount ?? 0;
  return new Intl.NumberFormat('en-US', {
    style: 'currency',
    currency: 'USD',
    minimumFractionDigits: 2,
    maximumFractionDigits: 2,
  }).format(value);
}
```

**formatDate implementation:**
Accept string | Date | null | undefined. Use date-fns parseISO for strings (parses as local time, NOT UTC — prevents timezone-shift bug where "2026-01-15" appears as January 14th). Use isValid() to check parse result. Wrap in try/catch to handle completely malformed strings. Return "Invalid date" for all failure cases — never throw.

Default format string is 'MMM d, yyyy' (e.g., "Jan 15, 2026"). Accept second parameter for alternate formats (e.g., "MMM yyyy" for chart axis labels).

```typescript
import { format, parseISO, isValid } from 'date-fns';

export function formatDate(
  date: string | Date | null | undefined,
  formatStr: string = 'MMM d, yyyy'
): string {
  if (!date) return 'Invalid date';
  try {
    const parsed = typeof date === 'string' ? parseISO(date) : date;
    if (!isValid(parsed)) return 'Invalid date';
    return format(parsed, formatStr);
  } catch {
    return 'Invalid date';
  }
}
```
