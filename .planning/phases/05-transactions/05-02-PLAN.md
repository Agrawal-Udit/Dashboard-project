    - Store role change triggers re-render (Zustand subscription handles this automatically)

  <action>
    Create directory `src/components/auth/` if it does not exist.
    Create `src/components/auth/RoleGate.tsx`.

    ```typescript
    import { useAppStore } from '../../store/store'

    type Role = 'Viewer' | 'Admin'

    interface RoleGateProps {
      allowedRoles: Role[]
      children: React.ReactNode
    }

    export function RoleGate({ allowedRoles, children }: RoleGateProps) {
      const role = useAppStore((s) => s.role)
      if (!allowedRoles.includes(role)) return null
      return <>{children}</>
    }
    ```

    Key decisions:
    - Single scalar selector `(s) => s.role` — no `useShallow` needed (scalar, not array/object)
    - `Role` type is defined inline here to avoid circular import — matches 'Viewer' | 'Admin' from uiSlice
    - Returns `null` (not empty fragment) when role is not allowed — so parent can use conditional rendering naturally
    - Under 20 lines total

    Do NOT add any Tailwind classes — this component is invisible in the DOM when children are shown.
